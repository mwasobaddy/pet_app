# Fix for Notifications Route Export Error

**Issue:** `Uncaught (in promise) SyntaxError: The requested module '/resources/js/routes/notifications/index.ts?t=...' does not provide an export named 'default'`

**Date Fixed:** April 23, 2026

## Root Cause

Wayfinder generated `resources/js/routes/notifications/index.ts` with only named exports (`index`, `read`) but the components were importing them as a default export:

```typescript
// Component import (expected default export)
import notificationRoutes from '@/routes/notifications'

// But the file only had named exports
export const index = (...)
export const read = (...)
```

## Solution Applied

Added a default export object at the end of `resources/js/routes/notifications/index.ts`:

```typescript
const notifications = {
    index: Object.assign(index, index),
    read: Object.assign(read, read),
}

export default notifications
```

This matches the pattern used by other Wayfinder-generated routes (e.g., `chat`, `matching/preferences`).

## Files Modified

- `resources/js/routes/notifications/index.ts` - Added default export at end of file

## How to Apply This Fix After Wayfinder Regeneration

If Wayfinder regenerates routes, this file will be overwritten. If the error returns:

1. Run: `php artisan wayfinder:generate`
2. Manually add at the end of `resources/js/routes/notifications/index.ts`:

```typescript
const notifications = {
    index: Object.assign(index, index),
    read: Object.assign(read, read),
}

export default notifications
```

3. Clear Vite cache: `rm -rf node_modules/.vite dist`
4. Restart dev server: `npm run dev`

## Alternative Solution (Not Applied)

Could modify how components import:

```typescript
// Instead of:
import notificationRoutes from '@/routes/notifications'

// Use named imports:
import { index, read } from '@/routes/notifications'
```

But current solution is better as it maintains consistency with other Wayfinder routes.

## Note

`resources/js/routes/` is in `.gitignore` because it's auto-generated by Wayfinder. This fix applies the same default export pattern used by other routes to ensure consistency.
